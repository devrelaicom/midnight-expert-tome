# Compact Runtime Errors

Errors thrown by `@midnight-ntwrk/compact-runtime` at contract execution time. All are `CompactError` instances (the base class) — catch `CompactError` to handle any runtime error generically.

**Source pin:** `LFDT-Minokawa/compact` @ `42b43a2` (main, 2026-05-01) / `runtime/` subtree / package version `0.16.0`.

## Base class

### CompactError

`CompactError` is the base class for all compact-runtime errors. Catch this type to handle any contract execution failure generically.

**Source:** `runtime/src/error.ts:21`.

---

## Assertion and type errors (`runtime/src/error.ts`)

### `failed assert: <message>`

Triggered when a Compact `assert` expression evaluates to `false` at runtime.

**Fix:** The `<message>` text is the string supplied to `assert` in the contract source — use it to locate the assertion and determine why the condition was not satisfied.

**Source:** `runtime/src/error.ts:33`.

### `type error: <who> <what> at <where>; expected value of type <type> but received <value>`

A runtime type mismatch in compiler-generated code. Most commonly: a TypeScript witness implementation returns a value whose shape does not match the type declared in the Compact contract.

**Fix:** Cross-check witness return types against the Compact-side declarations.

**Source:** `runtime/src/error.ts:43`.

## Version checks (`runtime/src/version.ts` — generated by `runtime/export-version.ss`)

### `Version mismatch: compiled code expects X.Y.Z, runtime is A.B.C`

The runtime version embedded in the compiled contract does not satisfy the installed `@midnight-ntwrk/compact-runtime` version.

**Fix:** Update `@midnight-ntwrk/compact-runtime` to a compatible version, or recompile the contract with a compiler whose target runtime matches the installed one.

**Source:** `runtime/export-version.ss:36` (generates into `runtime/src/version.ts`).

### `Maximum field mismatch: compiled code uses X, runtime uses Y`

The field-modulus constant baked into compiled code does not match the runtime's `MAX_FIELD`.

**Fix:** Recompile the contract with a compiler whose field modulus matches the installed runtime, or install a runtime version with the matching `MAX_FIELD`.

**Source:** `runtime/export-version.ss:42`.

## Cast errors (`runtime/src/casts.ts`)

### `range error at <src>: Field or Uint value <x> does not fit into <n> bytes`

`convertFieldToBytes` was asked to encode a value that overflows the requested byte width.

**Fix:** Reduce the source value, or cast to a wider Uint type with sufficient bytes.

**Source:** `runtime/src/casts.ts:34`.

### `range error at <src>: byte vector [...] exceeds maximum value <MAX_FIELD> of Field type`

`convertBytesToField` received a byte sequence whose integer value exceeds the field modulus.

**Fix:** Ensure the source bytes encode a value below `MAX_FIELD` before casting to `Field`.

**Source:** `runtime/src/casts.ts:47`.

### `range error at <src>: byte vector [...] exceeds maximum value <maxval> of target Uint type`

`convertBytesToUint` received bytes whose integer value exceeds the target Uint's `maxval`.

**Fix:** Cast to a wider Uint type, or validate/clamp the input bytes before casting.

**Source:** `runtime/src/casts.ts:63`.

## Type-validation errors (`runtime/src/compact-types.ts`)

These fire when a `CompactType.fromValue` or `CompactType.toValue` implementation receives a runtime value that does not match the expected shape.

### `expected JubjubPoint`

`CompactTypeJubjubPoint.fromValue` expected a value containing two field elements (x, y).

**Fix:** Confirm the upstream witness or value really produces a JubjubPoint.

**Source:** `runtime/src/compact-types.ts:109`.

### `expected MerkleTreeDigest`

`CompactTypeMerkleTreeDigest.fromValue` expected a value with a leading field element representing the digest.

**Fix:** Confirm the upstream value contains a single Field element where one is expected.

**Source:** `runtime/src/compact-types.ts:132`.

### `expected Field`

`CompactTypeField.fromValue` received an empty value where a Field was expected.

**Fix:** Inspect upstream witness or struct construction for an unexpected empty value at this position.

**Source:** `runtime/src/compact-types.ts:202`.

### `expected Enum[<=N]`

`CompactTypeEnum.fromValue` received an empty value where an Enum tag was expected.

**Fix:** Confirm the upstream value contains the byte sequence that encodes the Enum tag.

**Source:** `runtime/src/compact-types.ts:231`.

### `expected UnsignedInteger[<=N]`

Three call sites share this template — `CompactTypeEnum.fromValue` (overflow), `CompactTypeUnsignedInteger.fromValue` (missing value), and `CompactTypeUnsignedInteger.fromValue` (overflow). The message alone does not distinguish them; check the surrounding stack frame.

**Fix:** Confirm the upstream value is non-empty and within the target type's `maxValue` range. If overflow, cast to a wider Uint or reduce the source value.

**Source:** `runtime/src/compact-types.ts:238`, `:268`, `:275`.

### `expected <length>-element array`

`CompactTypeVector.toValue` received an array whose length does not match the declared Vector length.

**Fix:** Pad or truncate the input array to the exact declared length.

**Source:** `runtime/src/compact-types.ts:317`.

### `expected Boolean`

`CompactTypeBoolean.fromValue` received a value that is missing, has more than one byte, or whose single byte is not `0` or `1`.

**Fix:** Confirm the upstream value is a single byte equal to `0` (false) or `1` (true).

**Source:** `runtime/src/compact-types.ts:337`.

### `expected Bytes[<length>]`

`CompactTypeBytes.fromValue` received a value that is missing, or has length greater than the declared `Bytes` length.

**Fix:** Confirm the upstream value is a `Uint8Array` no longer than the declared `Bytes` length.

**Source:** `runtime/src/compact-types.ts:367`.

## Contract dependency / state errors (`runtime/src/contract-dependencies.ts`)

### `Expected <expected> but received <actual>`

The `expectedValueError` helper fires from type-guard assertions when a value does not match the asserted type (e.g., a `ContractAddress` or `CompactValue`).

**Fix:** Inspect the JSON-stringified `actual` value and match its shape to the asserted type.

**Source:** `runtime/src/contract-dependencies.ts:110`.

### `State map <stateMap> contains key without corresponding value`

During `compactADTDependencies` traversal a `StateMap` key was found with no associated value.

**Fix:** Inspect the state-map's structure for an inconsistent (key, value) pairing. If reproducible from a contract path, this indicates a state-construction bug worth filing upstream.

**Source:** `runtime/src/contract-dependencies.ts:345`.

### `State <stateValue> cannot be cast to a <expectedCastOutput>`

A `StateValue` could not be cast to the expected representation (map, array, or boundary Merkle tree).

**Fix:** Inspect the actual `StateValue` shape against the expected ADT. Versioning mismatches between contract code and on-chain state are a common cause.

**Source:** `runtime/src/contract-dependencies.ts:371`.

## Built-in size checks (`runtime/src/built-ins.ts`)

### `Expected 32-byte string` (in `persistentCommit`)

The `opening` parameter to `persistentCommit` was not exactly 32 bytes.

**Fix:** Pass an `opening` that is exactly 32 bytes — generate via a 32-byte hash if needed.

**Source:** `runtime/src/built-ins.ts:125`.

### `Expected 32-byte string` (in `degradeToTransient`)

The input to `degradeToTransient` was not exactly 32 bytes.

**Fix:** Pass an input that is exactly 32 bytes — typically the output of `persistentHash` or `persistentCommit`.

**Source:** `runtime/src/built-ins.ts:144`.

## Circuit context errors (`runtime/src/circuit-context.ts`)

### `'contractState' parameter <contractState> has unexpected type`

`createCircuitContext` / `createInitialQueryContext` received a `contractState` that is not a `ChargedState`, `ContractState`, or `StateValue`.

**Fix:** Pass a value of one of those three types from `@midnight-ntwrk/onchain-runtime-v1`. Construct via the runtime's documented constructors instead of an ad-hoc object literal.

**Source:** `runtime/src/circuit-context.ts:65`.

### Onchain-runtime error wrapped as `CompactError`

The query-circuit wrapper in `circuit-context.ts` catches any `Error` from `@midnight-ntwrk/onchain-runtime-v1` and rethrows it as `new CompactError(err.toString())`. The message text is whatever the upstream library produced — there is no fixed template.

**Fix:** Read the wrapped message itself; it comes from `@midnight-ntwrk/onchain-runtime-v1`, not from compact-runtime. Search the onchain-runtime source or a separate status-codes source for that specific message.

**Source:** `runtime/src/circuit-context.ts:230`.
