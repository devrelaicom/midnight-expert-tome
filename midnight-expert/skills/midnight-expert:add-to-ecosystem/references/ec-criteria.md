# Electric Capital Eligibility — Canonical Reference

For a project to appear in the Electric Capital Crypto Ecosystems report under Midnight, it must satisfy **four** GitHub-side requirements and include **one** verbatim attribution sentence in its README.

## GitHub requirements

1. The repository is hosted on GitHub.
2. The repository is **public**.
3. The repository has the GitHub topic **`midnightntwrk`**.
4. If the project uses Compact, the repository **also** has the GitHub topic **`compact`**.

## README attribution sentences

Use **one** of the three sentences below, **verbatim**. Do not customise the wording. Each sentence corresponds to one of three project categories.

### "Built on Midnight"

For dApps, contracts, or tooling that run directly on Midnight.

> "This project is built on the Midnight Network."

### "Integrates with Midnight"

For SDKs, infrastructure, wallets, or services that connect to Midnight.

> "This project integrates with the Midnight Network."

### "Extends Midnight"

For developer tooling, frameworks, or libraries that expand Midnight capabilities.

> "This project extends the Midnight Network with additional developer tooling."

## Required README format

The skill inserts the chosen sentence as a GitHub `> [!NOTE]` alert:

```markdown
> [!NOTE]
> This project is built on the Midnight Network.
```

Surrounded by blank lines on both sides. Placement is determined by `references/readme-placement.md`.

## Out of scope

Submitting the repo to the EC `crypto-ecosystems` GitHub repo is a separate step the user must do manually. This skill stops at "your project is eligible".
