# State Management

Midnight DApps manage two categories of state: public ledger state (on-chain,
visible to all participants) and private state (off-chain, known only to the
current user). The frontend combines these into a unified reactive stream
using RxJS and exposes the result to React components via hooks and contexts.

## RxJS combineLatest Pattern

The core pattern uses RxJS `combineLatest` to merge the on-chain state stream
from the indexer with the local private state. This produces a single
observable that emits whenever either source changes.

```typescript
import { combineLatest, map, retry, type Observable } from 'rxjs';
import type { ContractState } from '@midnight-ntwrk/midnight-js-types';

export interface DAppState<L, P> {
  ledgerState: L;
  privateState: P;
}

export function createStateObservable<L, P>(
  publicDataProvider: PublicDataProvider,
  contractAddress: ContractAddress,
  privateState$: Observable<P>,
  parseLedger: (state: ContractState) => L,
): Observable<DAppState<L, P>> {
  const ledger$ = publicDataProvider
    .contractStateObservable(contractAddress, { type: "latest" })
    .pipe(
      map((state) => parseLedger(state)),
      retry({ delay: 500 }),
    );

  return combineLatest([ledger$, privateState$]).pipe(
    map(([ledgerState, privateState]) => ({
      ledgerState,
      privateState,
    })),
  );
}
```

The `publicDataProvider.contractStateObservable()` method returns an RxJS
Observable that emits the contract's current public state whenever it changes
on-chain. Under the hood, this uses a WebSocket connection to the Midnight
indexer.

## The createStateObservable() Helper

The `createStateObservable()` function is the standard pattern used in the API
template. It takes four arguments:

1. **publicDataProvider** — The indexer-backed provider that streams on-chain
   state.
2. **contractAddress** — The deployed contract's address on the ledger.
3. **privateState$** — An observable of the local private state. For simple
   cases, this can be a `BehaviorSubject` that emits the current private state
   whenever it changes.
4. **parseLedger** — A function that converts raw `ContractState` bytes into a
   typed ledger structure. Typically this is `YourContract.ledger(state.data)`.

The helper returns `Observable<DAppState<L, P>>` where `L` is the typed ledger
state and `P` is the private state type. Components subscribe to this
observable and receive updates whenever either the on-chain state or the local
private state changes.

## Contract State Parsing

Raw contract state from the indexer is an opaque `ContractState` object
containing serialized bytes. Each compiled Compact contract generates a
`ledger()` function that deserializes these bytes into a typed structure:

```typescript
import { Contract as MyContract } from '@myproject/contract';

// Parse raw indexer state into typed ledger
const ledgerState = MyContract.ledger(contractState.data);

// Access typed fields
console.log(ledgerState.owner);
console.log(ledgerState.totalSupply);
```

The `ledger()` function is generated by the Compact compiler and lives in
the contract package. It is the only safe way to extract typed data from raw
contract state. Never attempt to parse the bytes manually.

## useContractState Hook

The `useContractState` hook is a generic React hook that subscribes to an
RxJS Observable and returns the current state along with any error:

```typescript
import { useEffect, useState } from 'react';
import type { Observable, Subscription } from 'rxjs';

export interface ContractStateResult<T> {
  state: T | null;
  error: Error | null;
}

export function useContractState<T>(
  observable: Observable<T> | null,
): ContractStateResult<T> {
  const [state, setState] = useState<T | null>(null);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    if (!observable) return;

    setError(null);

    const subscription: Subscription = observable.subscribe({
      next: (value) => {
        setState(value);
        setError(null);
      },
      error: (err) => {
        setError(err instanceof Error ? err : new Error(String(err)));
      },
    });

    return () => {
      subscription.unsubscribe();
    };
  }, [observable]);

  return { state, error };
}
```

**Key design decisions:**

- The hook accepts `Observable<T> | null` to handle the case where the
  observable is not yet available (e.g., before the contract is deployed or
  before providers are assembled).
- The `error` field captures both RxJS errors and connection failures.
- The cleanup function unsubscribes when the component unmounts or when the
  observable reference changes, preventing memory leaks and stale state.

## Auto-Reconnection with retry

WebSocket connections to the indexer can drop due to network instability. The
`retry({ delay: 500 })` operator automatically resubscribes to the source
observable after a 500ms delay when an error occurs:

```typescript
const ledger$ = publicDataProvider
  .contractStateObservable(contractAddress, { type: "latest" })
  .pipe(
    map((state) => parseLedger(state)),
    retry({ delay: 500 }),
  );
```

Without `retry`, a dropped WebSocket connection would terminate the observable
and the UI would stop receiving state updates. With `retry`, the observable
reconnects and resumes streaming. The 500ms delay prevents tight reconnection
loops during sustained outages.

For more sophisticated retry strategies, RxJS supports exponential backoff:

```typescript
import { timer } from 'rxjs';

retry({
  delay: (error, retryCount) => timer(Math.min(500 * 2 ** retryCount, 30000)),
})
```

## React Context Nesting Pattern

The standard context nesting order ensures that each provider has access to
everything it needs:

```tsx
<WalletProvider>
  <MidnightProvidersProvider>
    <ContractProvider>
      <App />
    </ContractProvider>
  </MidnightProvidersProvider>
</WalletProvider>
```

**WalletProvider** manages the Lace wallet connection. It stores the
`ConnectedAPI` and exposes `connect()` / `disconnect()` methods. This must be
the outermost provider because all other providers depend on the wallet
connection.

**MidnightProvidersProvider** calls `createProviders()` with the
`ConnectedAPI` from `WalletProvider` and stores the resulting 6 providers in
context. It depends on `WalletProvider` and must wrap any components that
interact with contracts.

**ContractProvider** (optional, per-contract) manages the deployed contract
instance and its state observable. It uses the providers from
`MidnightProvidersProvider` to deploy or find the contract, then creates the
state observable for components to consume.

## In-Memory vs Persistent Private State

**In-memory (browser):** Browser DApps use a `Map` for private state storage.
This state is lost when the user closes the tab or refreshes the page. For
most DApps this is acceptable because the relevant state can be reconstructed
from the public ledger. When the DApp reconnects to a deployed contract, it
reads the current public state and initializes a fresh private state.

```typescript
const privateStates = new Map<ContractAddress, PrivateState>();
```

**LevelDB (CLI/Node.js):** Integration tests and CLI tools use LevelDB for
persistent private state. This is provided by the SDK's
`levelPrivateStateProvider` and stores state on disk so it survives process
restarts.

```typescript
import { levelPrivateStateProvider } from '@midnight-ntwrk/midnight-js-level-private-state-provider';

const privateStateProvider = levelPrivateStateProvider({
  privateStateStoreName: 'private-states',
  signingKeyStoreName: 'signing-keys',
  accountId: walletAddress, // required: scopes LevelDB storage per account
  privateStoragePasswordProvider: () => process.env.PRIVATE_STATE_PASSWORD!,
});
```

The LevelDB-backed encryption is async under the hood (Web Crypto + PBKDF2 as of
midnight-js 4.1.0); the password from `privateStoragePasswordProvider` is subject to a
strict policy: it must be at least 16 characters and contain at least 3 of the 4
character classes (lowercase, uppercase, digits, symbols), or the SDK throws a
`PasswordValidationError`. A short value like a bare `process.env.PRIVATE_STATE_PASSWORD`
will fail at runtime unless it satisfies this policy.

Choose in-memory for browser DApps. Choose LevelDB for long-running processes
or integration tests where state must persist across runs.

## Derived State Pattern

Raw contract state often needs transformation before it is useful in the UI.
The derived state pattern computes UI-friendly values from the combined
ledger and private state:

```typescript
export interface BoardUIState {
  posts: Array<{
    id: string;
    content: string;
    isOwner: boolean;
  }>;
  canPost: boolean;
  totalPosts: number;
}

export function deriveBoardUIState(
  ledger: BoardLedgerState,
  privateState: BoardPrivateState,
): BoardUIState {
  const posts = ledger.posts.map((post) => ({
    id: post.id,
    content: post.content,
    isOwner: post.author === privateState.myAddress,
  }));

  return {
    posts,
    canPost: ledger.posts.length < ledger.maxPosts,
    totalPosts: ledger.posts.length,
  };
}
```

Apply the derivation inside the `map` operator of the combined observable,
or compute it in the consuming component. Keeping derivation logic in a
separate pure function makes it testable without any RxJS or React
dependencies.

```typescript
const uiState$ = createStateObservable(
  publicDataProvider,
  contractAddress,
  privateState$,
  parseLedger,
).pipe(
  map(({ ledgerState, privateState }) =>
    deriveBoardUIState(ledgerState, privateState),
  ),
);
```

## Subscription Cleanup

Always unsubscribe from observables in React `useEffect` cleanup functions.
Failing to do so causes memory leaks and can trigger state updates on
unmounted components:

```typescript
useEffect(() => {
  const subscription = observable.subscribe({
    next: (value) => setState(value),
    error: (err) => setError(err),
  });

  // Always clean up
  return () => subscription.unsubscribe();
}, [observable]);
```

If you are using the `useContractState` hook, cleanup is handled internally.
If you subscribe to observables manually (e.g., for side effects like
logging or analytics), ensure you manage the subscription lifecycle yourself.

Multiple subscriptions to the same observable should use RxJS `share()` or
`shareReplay(1)` to avoid creating duplicate WebSocket connections to the
indexer:

```typescript
const shared$ = publicDataProvider
  .contractStateObservable(contractAddress, { type: "latest" })
  .pipe(shareReplay(1));
```

This ensures that multiple subscribers share a single underlying WebSocket
connection, reducing load on the indexer and preventing race conditions in
state updates.
